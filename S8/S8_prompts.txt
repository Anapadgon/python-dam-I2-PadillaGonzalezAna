Explicación:
Lo que hace es gestionar las ventas diarias de una panadería de Mercadona. Permite añadir registros con el día, 
tipo de pan y cantidad vendida, buscar por tipo de pan, mostrar todos los datos, calcular la media de ventas y 
averiguar cuál fue el pan más vendido, controlando errores como campos vacíos, datos duplicados o valores incorrectos.

Después de los errores comentados en clase he modificado el programa para que la lista produccion no sea una variable global. 
Ahora, la lista se crea dentro de la función menu() y se pasa como parámetro a cada función. Cada función recibe la lista, 
la utiliza (añadiendo, buscando o mostrando datos) y la devuelve actualizada con return produccion. De esta forma, el programa
sigue funcionando igual, pero con una mejor estructura y sin usar variables globales.

En cuanto a diferencias en una estructura inversa:
- En una lista de diccionarios, cada elemento guarda toda la información de un solo registro (día, tipo de pan, cantidad).
- Es útil cuando quieres recorrer los registros uno a uno o añadir nuevos datos fácilmente.
- En un diccionario de listas, los datos están agrupados por una categoría (por ejemplo, todos los panes de un día o todas las ventas de un tipo).
- Es más útil cuando quieres consultar rápidamente qué se vendió en un día concreto o agrupar información.
